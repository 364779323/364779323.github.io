JDBC

JDBC的概念

JDBC(Java Database Connectivity) 让我们可以使用java语言来操作数据库
如果操作不同的数据库需要学习不同的操作方法，那样就大大增大了程序员的学习成本，于是官方定义了一套可以操作所有关系型数据库的规则，即接口。各个数据库厂商去实现这套接口，提供数据库驱动jar包。我们可以使用这套接口编程，真正执行的代码是驱动jar包中的实现类。

JDBC的快速入门

基本步骤

  1. 导入驱动jar包
  2. 注册驱动 Class.forname(“com.mysql.jdbc.Driver”) C要大写
    (如果是mysql8.0版本的驱动名变了 为com.mysql.cj.jdbc.Driver 本文使用的mysql版本为5.7)
  3. 获取数据库链接对象connection
  4. 定义sql语句
    1. 获取执行sql操作的对象statement（为了防止sql注入 所以基本是使用PreparedStatement）
    2. 执行sql，接受返回结果
    3. 处理结果
    4. 释放资源

来讲一讲在上面步骤中出现的一些类

DriverManager:驱动管理对象

static void registerDriver(driver):注册与给定的驱动程序
com.mysql.jdbc.Driver类中存在静态代码块 会调用registerDriver来注册驱动
mysql5后可以不用自己注册驱动 但还是最好写上 这样代码就能向下兼容

获取数据库链接:DriverManager.getConnection():三个参数

  1. url jdbc:mysql://ip地址(域名):端口号/数据库名称
    tips:如果是访问本机mysql数据库，且端口为3306 可以写成jdbc:mysql:///数据库名称
  2. username

  3. password

    Connection:数据库链接对象

    功能

  4. 获取执行sql操作的对象statement

    createStatement
PreparedStatement preparedStatement(String sql)

  5. 管理事务:

    开启事务: void setAutoCommit(boolean autoCommit) 
提交事务:commit()
回滚事务:void rollback()

Statement:执行sql的对象(静态)

  1. boolean excute(String sql);可以执行任意的sql

  2. int excuteUpdate(String sql);执行DML(insert update delete)语句、DDL语句(create,alter,drop)(不常用) 返回值为影响的行数 可以通过这个数来判断dml语句是否执行成功 如果大于0则执行成功

    tips:1000更新成1000不算在影响的行数内

  3. ResultSet excuteQuery(String sql) 执行DQL(select)语句

ResultSet:结果集对象 封装查询的结果 列的编号从1开始

ResultSet也是资源 也需要关闭
有个游标 一开始指向表头 需要手动resultset.next()将游标下移 然后调用getXXX方法来获取我们需要的值(参数可以用列的编号或者列的名称)

正确的使用方式：

步骤：

  1. 游标向下移动一行
    1. 判断是否有数据(next方法会返回一个boolean值)
  2. 如果有 则获取数据

PreparedStatement:执行sql的对象(预编译）

sql注入问题:在拼接sql时，有一些sql的特殊关键字参与字符串的拼接 会造成安全性的问题使用statement会面临遭遇sql注入的问题 如输入用户名随便 输入密码a’ or ‘a’=’a
例如sql语句为
select * from USER where username=’qqqqqq’ and PASSWORD=’a’ or ‘a’=’a’
即可登录成功,因此我们可以使用PrepareStatement对象来解决 预编译sql使用参数?作为占位符

步骤:

  1. 定义sql时参数使用?作为占位符 select * from user where username = ? amd password = ?
  2. 获取PrepareStatement对象 connection.prepareStatement(String sql) !!在此时将sql语句传入
  3. 给？赋值
    方法 setXxx(参数1，参数2) (例如setString setInt 等等)
    参数1:？的位置编号 从1开始
    参数2:？的值
  4. 执行sql，接受返回结果 （无需传入sql语句）

释放资源！

计算机的资源是有限的 如果不及时的释放资源 有可能会造成内存泄漏 比如1g的内存用着用着就只剩500m了 这是非常不好的事情
所以我们将释放资源的语句写在finally语句中 并先进行判断是否为NULL 从而避免空指针异常

抽取工具类

写了一些小demo后我们会发现 会有大量的重复代码 我们可以把这部分抽取成一个工具类JdbcUtils 从而简化书写
我们抽取出来的JdbcUtils主要有两个功能

  1. 获取连接

    不能像原来一样获取 那样会造成工具类的不通用

    如果使用传参 过于麻烦 并没有做到真正意义上的简化
    解决方法：
    使用配置文件来存储数据库url username password等信息

    jdbc.properties 文件的读取 只需读取一次就能拿到这些值 所以使用静态代码块
    使用properties集合类来读取properties配置文件
    加载文件时如果写绝对路径 后期更改路径过于麻烦 所以使用CLassLoader类加载器来获取路径
    1.new一个properties对象
    2.new一个ClassLoader对象
    3.使用classloader.getResourse获取url地址 再调用getPath即可得到路径

  1. 释放资源
    使用重载书写
    因为当进行增删改的时候 需要释放statement connection两个对象
    进行查询的时候 需要释放resultset statement connection三个对象

JDBC事务

事务：一个包含多个步骤的业务操作，如果这个业务操作被事务管理，则这多个步骤要么同时成功，要么同时失败
操作：

  1. 开启事务 setAutoCommit(boolean autoCommit) 调用此方法设置参数为false，即开启事务（默认是关闭的）
  2. 提交事务 commit()
  3. 回滚事务 rollback()
    使用connection对象来管理事务
    在执行sql前开启事务 当所有sql都执行完提交事务 在catch中回滚事务

如果我们在preparedStatement1.executeUpdate();和preparedStatement2.executeUpdate();之间添加一行int a= 3/0;众所周知 会抛出异常
此时 就会发生事务回滚 如果不再catch中写上rollback() 则会出现上面那条成功执行而下面那条未能成功执行的状况 很显然 这不是我们想要看到的
